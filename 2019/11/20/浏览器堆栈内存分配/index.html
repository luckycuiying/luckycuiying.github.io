浏览器堆栈内存分配

Nov 20, 2019点击

什么是堆栈内存以及分配？

浏览器在执行程序时形成两个虚拟的内存 用来存储数据和提供代码执行环境。

  • 基本类型的数据类型存储到栈内存（stack)中， 引用数据类型存储到栈(heap)内存中

    • heap 堆内存数据没有结构
    • stack 栈是有结构的，有次序的

  • 对象的定义

    • 去堆内存开辟一个空间
    • 把对象中的键值对放在堆内存中
    • 把堆内存中king见地址赋值给栈内存的变量
      变量和对象的指针存储到栈内存， 当在栈内存中使用这个对象，通过指针访问到值

浏览器的堆内存回收

  • 目的： 性能优化
  • 常见的浏览器堆内存回收方法
    • 标记清除
      • 给每一个空间地址记上标记
      • 筛选过滤环境的变量或者被应用的变量去除标记
      • 加上标记的就视为被将要删除的变量
      • 浏览器对有标记的进行回收，释放内存
    • 引用计数
      • 对变量被应用一次就会在这个变量技术+1
      • 如果不在应用，那么计数会默认-1
      • 该变量计数为0时，浏览器要回收该块内存

目前浏览器IE， 火狐， 谷歌 opera大都采用标记清除的方式

  • 垃圾回收器： GC 每个一段时间执行一次

内存泄露

  • 全局的变量不能被回收， 因为回放在window 这个对象中
  • 闭包引发的内存泄露
